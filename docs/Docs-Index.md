## 4. 建議閱讀順序

如果你是第一次打開這個 repo，建議依照下面順序閱讀：

1. 先看快速啟動，知道怎麼把專案跑起來。
2. 再看功能地圖，知道每個功能放在哪裡。
3. 接著看第一條黃金路徑，實際跑一次建立商品、補庫存、下單與通知。
4. 最後再依角色進入更深的設計、維運或擴充文件。
